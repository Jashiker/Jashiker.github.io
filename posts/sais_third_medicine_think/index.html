问题

镜像问题

Task1的时候还可以使用aliyun云镜像服务进行构建，但是到了做Task2的时候，就不能使用aliyun云镜像服务进行构建。

无奈只能尝试本地构建，根据云镜像服务的说明使用Linux主机进行docker的构建，首先将代码仓库用如下命令拉下来:

git clone https://codeup.aliyun.com/67fc70cbfb34110a021fb833/sais_medicine.git

然后进入到sais_medicine目录下使用docker构建的命令进行镜像的构建:

sudo docker build -t sais_medicine:v5 .

查看镜像的id，以便后续打标签推送

sudo docker images

输出如下结果： 如果docker运行没有问题就可以进行打标签操作了，按照云镜像服务的说明，打标签：

sudo docker tag 8e232b63ce72 crpi-mesc8odxqa2miyzr.cn-hangzhou.personal.cr.aliyuncs.com/sais_thrid_jashacker/sais_medicine:v5

推送到云镜像服务上：

 sudo docker pull crpi-mesc8odxqa2miyzr.cn-hangzhou.personal.cr.aliyuncs.com/sais_thrid_jashacker/sais_medicine

torch-cluster

进阶算法使用到了python的torch-cluster，python的torch-cluster安装花费了挺多时间的，经常安装不成功或者安装成功了版本不符合调用失败。

针对安装不成功或者安装成功了版本不符合调用失败这个问题，根据pypi官网https://pypi.org/project/torch-cluster/的说明进行安装可以了，使用如下命令进行安装：

pip install torch-cluster -f https://data.pyg.org/whl/torch-2.1.0+${CUDA}.html

可以根据torch的版本和系统cuda版本进行修改，比如，torch-2.0.0，cuda-1.1.8的环境使用如下命令进行安装：

pip install torch-cluster -f https://data.pyg.org/whl/torch-2.0.0+cu118.html

如果输入正确无误安装应该很快就可以完成。

提交

提交的时候和Task1不太一样，由于使用了新算法，新模型之前的run.py进行加载模型时容易出现错误。

修改run.py使其符合新算法，能够成功加载新算法生成的模型，同时也要修改DockerFile的内容使其成功安装python的torch-cluster。
